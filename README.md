# 📊 Reviews: Агрегация и Аналитика

Система автоматизированного сбора, обработки и анализа отзывов пациентов сети клиник «Эмирмед». Проект представляет собой решение на базе Google Apps Script, интегрированное с Google Sheets, предназначенное для формирования единой базы отзывов из внешних источников и генерации аналитических отчетов.

## 🎯 Назначение

Проект решает задачу централизации информации о качестве услуг, объединяя данные с порталов записи (DOQ) и карт (2GIS) в единый интерфейс. Система автоматизирует рутинные процессы: от сбора данных и идентификации сотрудников до построения сравнительных отчетов и дашбордов.

## ⚙️ Основные возможности

*   **Мультивендорный сбор:** Параллельная выгрузка отзывов из API систем DOQ и 2GIS с фильтрацией по филиалам и периодам.
*   **Умная идентификация (NLP):** Автоматическое извлечение имен сотрудников из текста отзывов 2GIS с помощью AI и дальнейшая сверка найденных имен с внутренней базой врачей через Algolia.
*   **Аналитика и Визуализация:**
    *   Построение сводных таблиц с расчетом лояльности и индекса NPS.
    *   Генерация интерактивных дашбордов с графиками динамики и семантическими облаками слов.
    *   Формирование ежемесячных аналитических отчетов в формате Google Docs.
*   **Управление данными:** Инструменты для массового удаления данных за период и пересчета статистики.

## 🔄 Логика работы системы

### 1. Сбор данных (Fetch)
*   Система авторизуется во внешних системах.
*   Запрашивает списки отзывов по каждому активному филиалу.
*   Фильтрует отзывы по заданному пользователем периоду.
*   Записывает сырые данные на соответствующие листы таблицы.

### 2. Обработка и Нормализация
*   Для отзывов, где имя сотрудника не указано явно (источник 2GIS), запускается AI-модуль.
*   AI выделяет имена из текста, которые затем проверяются через поисковый движок Algolia по базе врачей DOQ.
*   Найденные совпадения автоматически проставляются в таблицу.

### 3. Отчетность
*   Агрегация данных по уровням: `Сеть -> Регион -> Филиал -> Врач`.
*   Расчет KPI (лояльность, средний балл).
*   Выявление лидеров и аутсайдеров.

## 🛠️ Настройка окружения

Для работы системы необходимо задать параметры авторизации и справочники:

*   **Токены доступа:** Учетные данные для API DOQ (разные токены для городов) и логин/пароль для 2GIS.
*   **Ключи поиска:** ID и API Key для сервиса Algolia (используются для поиска врачей).
*   **Справочник филиалов:** Таблица соответствия названий листов таблицы внутренним ID систем (DOQ ID, 2GIS ID) и привязка к регионам.

## 🚀 Использование

Управление системой осуществляется через пользовательское меню **"⚙️ Меню Отзывов"** в интерфейсе таблицы Google Sheets. Процесс работы строится последовательно:

1.  **Сбор:** Запуск задачи сбора отзывов, выбор нужных филиалов и периода.
2.  **Обработка:** Запуск нормализации имен (после того как AI-формулы отработают в ячейках).
3.  **Анализ:** Формирование сводных таблиц или генерация дашборда.
4.  **Отчетность:** Создание итогового документа для руководства.

> 💡 **Примечание:** Система поддерживает гибкую фильтрацию, позволяя собирать данные как за всё время, так и за конкретный интервал, а также массово удалять устаревшие записи для поддержания актуальности базы.
